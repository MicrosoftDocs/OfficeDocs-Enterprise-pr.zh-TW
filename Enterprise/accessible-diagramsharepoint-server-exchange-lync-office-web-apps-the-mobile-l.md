Exchange Client Access Server (CAS) 主控的 Exchange ActiveSync 服務。它可讓您同步處理您的電話或平板電腦與 Exchange 2013 間的資料。 
  
### <a name="lync-deployment"></a>Lync 部署

為一般的工作流程，外部的行動裝置將 HTTPS 取得要求傳送至外部 Lync URL 範例： LyncDiscover.contoso.com。反向 proxy 伺服器，將要求透過連接埠 443 轉送到前端伺服器 （或伺服器集區） 所收到的要求。反向 proxy 伺服器必須具備的發行規則的 LyncDiscover.contoso.com。也建議防火牆和反向 proxy 伺服器架設在不同的電腦上。
  
在前端伺服器、 自動探索服務擷取使用者的主集區的 Web 服務 Url，並透過反向 proxy 伺服器的行動裝置會傳回該資訊。自動探索服務可讓行動裝置能夠尋找 Lync Server Web 服務而不需使用者手動輸入 Url。
  
如果行動裝置會防火牆內部登入、 HTTPS 要取得的命令會直接傳送至 Lync 前端集區 LyncDiscoverInternal.com 等。連接埠 443，溝通行動裝置使用自動探索服務的相同連接埠上收到要求。
  
您必須以支援行動性設定兩個的 DNS 記錄： 一個適用於 LyncDiscover.contoso.com，一個用於 LyncDiscoverInternal.com。應該只從防火牆; 外部存取的外部 FQDN應僅可從防火牆內部存取的內部 FQDN。
