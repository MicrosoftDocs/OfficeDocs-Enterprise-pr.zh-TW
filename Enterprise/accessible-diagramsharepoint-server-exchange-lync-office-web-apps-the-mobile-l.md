## <a name="on-premises-and-cloud-based-mobile-deployments"></a>內部部署和雲端式行動部署

拓撲，以下為 SharePoint Server，Lync 及 Exchange 不會說明實作這些平台的慣用的方法。 它們只會提供範例中，為拓撲會不同根據唯一的網路需求及安全性考量。
  
### <a name="sharepoint-deployment"></a>SharePoint 部署

隨附圖顯示 SharePoint Server 2013 的外部網路部署。 我們建議裝置安全地連線到 SharePoint 2013 web 前端伺服器使用的選項，例如安全通訊端層 (SSL) 或 VPN 閘道。
  
SharePoint 可以使用 TCP 連接埠 443 (SSL) 加密伺服器之間之通訊裝置及反向 proxy。 來自網際網路的外部存取，此連接埠需要開啟任何防火牆或路由器上的輸入和輸出流量。 
  
當行動裝置瀏覽器存取網站從 web 前端伺服器時，IIS 會進行 http 要求。 Http 要求中，[USERAGENT 欄位會包含提出要求的行動裝置瀏覽器的相關資訊。 此資訊可用於--現成的新型檢視中或使用裝置通道功能的自訂網站實作的瀏覽器重新導向決策。
  
SharePoint 2013 的 Office Web Apps 脫離安裝，以及現在是獨立的產品。 若要提供 Office Web Apps，包括行動裝置，用戶端存取您要設定 SharePoint 2013 使用 Office Web Apps Server 安裝。 您可以使用下列 SharePoint 2013 管理命令介面 cmdlet 來這麼做：
  
```
New-SPWOPIBinding -ServerName <ServerName>, where <ServerName> is the FQDN name of the URL that you set for the internal and external URLs.
```

### <a name="exchanage-deployment"></a>Exchanage 部署

裝置連線到內部部署 Exchange 用戶端存取伺服器主控應用程式如 Microsoft Exchange ActiveSync、 Outlook Web app，並且也提供 Exchange 自動探索服務。 我們建議您使用外部裝置與 Exchange Client Access Server 之間的 SSL 加密。
  
Exchange 自動探索和 Exchange ActiveSync 等裝置流量被處理透過連接埠 443 (HTTPS)。 Direct Push 功能在 Exchange 2013 會保留在行動裝置目前透過行動電話或無線網路連線。 新內容可同步時，就會通知行動裝置。 請務必在您的組織和電訊廠商中的所有防火牆都支援長期 HTTPS 要求。 根據預設 Direct Push 會輪詢 15 分鐘內任何資料夾中的任何變更，並更新裝置設定。 您可能必須增加防火牆的逾時時間值，如果加以落下方的預設 15 分鐘 Direct Push 設定。
  
Exchange Client Access Server (CAS) 主控的 Exchange ActiveSync 服務。 它可讓您同步處理您的電話或平板電腦和 Exchange 2013 之間的資料。 
  
### <a name="lync-deployment"></a>Lync 部署

為一般的工作流程，外部的行動裝置將 HTTPS Get 要求傳送至外部 Lync URL 範例： LyncDiscover.contoso.com。 反向 proxy 伺服器，要求轉送給連接埠 443 上的前端伺服器 （或伺服器集區） 所收到的要求。 反向 proxy 伺服器必須擁有 LyncDiscover.contoso.com 發行規則。 我們也建議防火牆和反向 proxy 伺服器會裝載在不同的電腦上。
  
在前端伺服器上，自動探索服務擷取使用者的主集區的 Web 服務 Url，並透過反向 proxy 伺服器與行動裝置會傳回該資訊。 自動探索服務可讓行動裝置尋找 Lync 伺服器 Web 服務，而不需使用者手動輸入 Url。
  
如果行動裝置防火牆內部登入，HTTPS Get 命令都會直接傳送到 Lync Front End Pool LyncDiscoverInternal.com 等。 在 [連接埠 443，溝通行動裝置使用自動探索服務的相同連接埠上收到要求。
  
您必須設定兩個 DNS 記錄，以支援行動性： 一個用於 LyncDiscover.contoso.com，一個用於 LyncDiscoverInternal.com。 應該只會從外部防火牆; 可以存取的外部 FQDN內部 FQDN 應僅能從防火牆內存取。
