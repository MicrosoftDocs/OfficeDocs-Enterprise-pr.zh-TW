## <a name="lync-presence-in-office-2013-outlook-web-app-and-sharepoint-server-tab"></a>Office 2013、 Outlook Web App 與 SharePoint Server] 索引標籤中的 Lync 顯示狀態

### <a name="lync-server-as-the-authoritative-source-of-presence-information"></a>Lync Server 做為目前狀態資訊的授權來源

使用 Lync 顯示狀態資訊，您可以有一致的檢視之間 Lync、 Outlook 和 SharePoint 的目前狀態資訊。Outlook 查詢直接從 Lync，在本機上已安裝 Outlook 的同一部電腦上的目前狀態資訊。當使用者在 SharePoint Server 中檢視的目前狀態資訊時，是由 Lync 本機電腦上查詢之目前狀態資訊。
  
用戶端產品： 
  
- Outlook 2013 
    
- Lync 2013 
    
伺服器產品： 
  
- Exchange Server 2013 
    
- Lync Server 2013 
    
- SharePoint Server 2013 
    
 **運作方式**
  
只要使用者本機電腦上安裝 Lync 2013，則 Outlook 與 SharePoint Server 自動顯示使用者的目前狀態資訊。 
  
Outlook Web App 使用者的 Exchange CAS 查詢即代表使用者的目前狀態。 
  
 **有兩個隨附的圖表**
  
第一個顯示如何使用者登入 Outlook Web App，然後交換目前狀態資訊的查詢 Lync Server。 
  
1. 使用者登入 Outlook Web app。用戶端電腦存取 Exchange Client Access Server (CAS) 透過 HTTPS/443 和也會呼叫 Exchange Mailbox Server 的內部 Exchange 通訊。 
    
2. 使用者登入其 Exchange 2013 信箱和 Exchange CAS over SIP 查詢的目前狀態資訊 Lync Server / MTLS:5061。 
    
如需詳細資訊，請參閱[整合 Microsoft Lync Server 2013 與 Microsoft Outlook Web App 2013](https://go.microsoft.com/fwlink/?LinkId=313522)。
  
第二個圖顯示如何 Outlook 與 SharePoint Server 使用 Lync 2013 來顯示使用者的目前狀態資訊。 
  
1. 使用者登入 Lync 2013 over SIP / TLS:5061，呼叫 Lync server。 
    
2. A.使用者登入 Exchange 2013 信箱透過 Office 2013 中的 Outlook。用戶端電腦存取 Exchange Client Access Server (CAS) 透過 HTTPS/443 和它也會呼叫 Exchange Mailbox Server 的內部 Exchange 通訊。 
    
3. A.outlook 呼叫 Outlook 擷取目前狀態資訊的同一部電腦上安裝 Lync。 
    
4. B.user 連接至 SharePoint MySite 透過 HTTP 或 HTTPS，呼叫的 SharePoint 伺服器。 
    
5. B.Internet Explorer 呼叫 Lync，在瀏覽器的同一部電腦上已安裝擷取目前狀態資訊。 
    
## <a name="voicemail-tab"></a>語音信箱] 索引標籤

### <a name="exchange-um-is-the-voicemail-system-for-lync-server"></a>Exchange UM 是 Lync Server 的語音信箱系統

語音信箱可讓來電者留下語音信箱給 Lync 使用者使用 Exchange 整合通訊 (UM)。 
  
用戶端產品： 
  
- Lync 2013 
    
- PSTN 裝置 （PBX、 行動、 POTS） 
    
伺服器產品： 
  
- Exchange Server 2013 
    
- Exchange Server 2013 
    
 **運作方式**
  
當呼叫未受話者在任何受話者的作用中的端點上接聽時 Lync Server 將通話路由傳送至語音信箱在 Exchange UM （亦即 Exchange 信箱伺服器）。 
  
隨附的圖顯示的兩種情況的電話路由： 
  
- 來電者起始使用 Lync 2013 的通話。 
    
- 來電者起始使用 PSTN 裝置 （PBX、 行動、 POTS） 的通話。 
    
來電者會使用 Lync 2013 的通話： 
  
1. 發話者的起始受話者使用 Lync 2013 呼叫。通話已啟動並傳送到 Lync Server。 
    
2. 將來電路由傳送至受話者 Lync 主伺服器。 
    
3. Lync Server 鈴響受話者在 Lync 2013 作用中的端點。 
    
4. 當未接聽來電時將來電路由傳送至語音信箱 (Exchange UM) 上 Exchange CAS （呼叫路由器）。 
    
來電者會使用 Lync 2013 的通話： 
  
1. 發話者 B 撥打使用 PSTN 的受話者的電話號碼。 
    
2. PSTN 通話路由傳送來自 IP 閘道到中繼伺服器，這是 Lync server。 
    
3. 中繼伺服器 routs 受話者 Lync 主伺服器的呼叫。 
    
4. Lync Server 鈴響受話者在 Lync 2013 作用中的端點。 
    
5. 當未接聽來電時將來電路由傳送至語音信箱 (Exchange UM) 上 Exchange CAS （呼叫路由器）。
