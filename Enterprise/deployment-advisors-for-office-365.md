# <a name="deployment-advisors-for-office-365-services"></a>Office 365 服務的部署建議

Office 365 的部署建議可讓您自行引導式的支援 for 設定 Office 365 為您的組織。 只要選取安裝輔助線或顧問，然後登入 Office 365。 

當您選取的功能與您想要部署的選項，顧問建置自訂符合您需求的逐步安裝計劃。 您會收到一套完整的指示、 視訊、 參考文章，以及指令碼。 一些建議有將會變更某些您的設定，而在其他部分顧問變更您的設定和資料的顧問，能引導您的自動化。 任何時候，即使在規劃期間或之後您已經設定好的服務，若要深入了解 Office 365 的功能和選項，您可以使用這些建議。
  
## <a name="windows-10-with-office-365"></a>透過 Office 365 的 Windows 10

[Microsoft 365 部署建議程式](https://aka.ms/microsoft365setupguide)
  
Microsoft 365 是完整的智慧型解決方案，其中包含 Office 365、 Windows 10 和 Enterprise Mobility + Security。 Microsoft 365 讓所有人發揮創意並安全地，在一起。 使用此顧問來設定 Windows 10 裝置，或將使用者的電腦為 Windows 10 升級選擇性部署 Office 應用程式、 Windows Analytics 和 Windows Defender 進階威脅防護 （僅限 Microsoft 365 企業版 E5 方案）。


## <a name="mail-migration-and-protection"></a>郵件移轉和保護

### <a name="prepare-your-environment"></a>準備您的環境 
[準備您的環境指南](https://go.microsoft.com/fwlink/?linkid=2005213)是可以從這裡開始。 請告訴我們您的部署目標，以確定您新增網域、 建立使用者，並指派適當的順序中的授權。 這是特別重要，如果您計劃要移轉電子郵件，或在設定混合式部署。 

### <a name="exchange-migration-advisor"></a>Exchange 移轉顧問
[Office 365 郵件移轉顧問](https://aka.ms/office365setup)可協助您將信箱從您目前的郵件系統移至 Exchange Online 在 Office 365 中，自動化的工具與逐步指引。 我們會建議您根據您目前的郵件系統信箱數目您想要移轉，以及如何規劃來管理使用者和使用者存取貴組織的最佳移轉路徑。
  
[將 Gmail 連絡人和行事曆助理](https://aka.ms/gmailcontactscalendar)提供將您的 Gmail 連絡人和 Google 行事曆項目移轉至 Office 365 的逐步指示。 當您將使用者的 Gmail 信箱移轉到 Office 365 時，移轉電子郵件訊息，但連絡人和行事曆項目不是。 本指南提供的步驟來匯入至 Office 365 的 Gmail 連絡人和 Google 行事曆項目。
  
### <a name="exchange-online-protection"></a>Exchange Online Protection
Microsoft [Exchange Online Protection (EOP)](https://aka.ms/EOPguidance)是雲端式電子郵件篩選服務，可協助組織抵禦垃圾郵件和惡意程式碼，同時也包括預防組織通訊原則違規發生的功能。

## <a name="security-and-identity"></a>安全性和身分識別

### <a name="azure-active-directory-connect-azure-ad-connect-advisor"></a>Azure Active Directory Connect (Azure AD Connect) 顧問
[Azure AD Connect 顧問](https://aka.ms/aadconnectpwsync)會示範如何新增至 Office 365 的使用者帳戶資訊，而不需要以手動方式建立每位使用者的逐步指南。 它也會顯示您如何設定密碼雜湊同步處理或通過驗證，讓使用者可以登入他們的電子郵件和您的網域使用相同的密碼。 如果您選擇設定 [同盟登入與 AD FS，本指南也會提供您步驟在新伺服器上或現有的 Windows Server 2012 R2 伺服器陣列中的 AD FS 的部署。 Azure AD Connect 取代較舊版本的身分識別整合工具，例如 DirSync 和 Azure AD 同步處理，並主要用於 for Office 365 中將使用者及其他資料新增至 Azure Active Directory。
  
### <a name="azure"></a>天藍白
[Azure AD 基本安裝指南](https://aka.ms/azureadbasic)可協助您設定功能，如以群組為基礎的存取管理、 自助密碼重設雲端應用程式和 Azure Active Directory 應用程式 Proxy 發佈的內部 web 應用程式。
  
[Azure AD Premium 設定指南](https://aka.ms/aadpguidance)可協助您啟用 Azure Active Directory Premium (Azure AD Premium)，提供身分識別管理功能，像是多重要素驗證單一登入 (SSO)，裝置註冊的許多功能自助式密碼以及群組管理和安全性監視。
  
### <a name="verify-your-domain"></a>Verify your domain
[確認 Office 365 中的網域](https://aka.ms/verifyyourdomaino365)顧問可協助您自訂 Office 365 新增您自己的網域名稱 （也就是您網站的名稱）。
